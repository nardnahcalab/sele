================================================================================
KEY FEATURES
================================================================================

1. Skill Protocol (sele.interfaces.Skill)
   - initialize(ctx) - Called once before loop starts
   - before_step(step_index, memory) - Called before each model step
   - after_step(step_index, response, tool_results) - Called after each step
   - on_loop_end(final_text, total_steps) - Called when loop terminates

2. Configuration Schema (SkillsConfig)
   - enabled: bool - Enable/disable skills
   - skills: list[str] - List of skill names
   - breadth: int - Number of parallel branches
   - depth: int - Maximum reasoning depth
   - context_window: int - Max context size
   - context_compression: bool - Enable compression
   - loop_strategy: str - Override loop implementation
   - skill_settings: dict - Per-skill configuration

3. Built-in Skills
   - reflexion: Self-reflection and iterative improvement
   - context_manager: Context window and compression management

4. Registry Integration
   - In-tree registration via @skill decorator
   - Out-of-tree registration via entry points
   - Lazy loading of skills on demand

5. Profile Configuration
   - YAML-based configuration
   - Per-skill settings
   - Global settings (breadth, depth, context_window)

USAGE EXAMPLES
================================================================================

List Available Skills:
  $ sele skills

Run with Reflexion:
  $ sele run "your task" -p reflexion-ollama

Run with Context Management:
  $ sele run "long task" -p context-managed-ollama

Run with Combined Skills:
  $ sele run "complex task" -p skills-combined-ollama

Create Custom Skill:
  from sele import skill
  from sele.skills import BaseSkill
  
  @skill("my_skill")
  class MySkill(BaseSkill):
      name = "my_skill"
      
      def initialize(self, ctx):
          print("Initializing")
      
      def before_step(self, step_index, memory):
          print(f"Step {step_index}")
      
      def after_step(self, step_index, response, tool_results):
          print("Step completed")
      
      def on_loop_end(self, final_text, total_steps):
          return final_text + f"\n[{total_steps} steps]"

Profile Configuration:
  loop:
    kind: tool_loop
    max_steps: 50
    skills:
      enabled: true
      skills: [reflexion, context_manager]
      breadth: 1
      depth: 1
      context_window: 8000
      skill_settings:
        reflexion:
          reflection_threshold: 3
          max_reflections: 2
        context_manager:
          max_context_chars: 8000
          compression_ratio: 0.5

================================================================================
ARCHITECTURE HIGHLIGHTS
================================================================================

1. Pluggable Design
   - Skills follow sele's pluggable architecture pattern
   - Easy to add custom skills without modifying core code
   - Registry-based discovery and instantiation

2. Lifecycle Hooks
   - Skills integrated at four key points in loop execution
   - Minimal overhead when skills are disabled
   - Skills can inspect and modify context

3. Configuration
   - YAML-based configuration for ease of use
   - Per-skill settings via skill_settings dict
   - Global settings for breadth, depth, context_window

4. Loop Integration
   - Skills initialized before loop starts
   - Hooks called automatically by LoopBase
   - Skills can override loop strategy

5. Composition
   - Multiple skills can be combined
   - Skills called in order
   - Each skill can modify context or inject prompts
